Research Logic

Research Logic: From Online Decisions to Non-Stationary Bandits

This project is not just a collection of performance curves. It answers a layered question: how Thompson Sampling balances exploration and exploitation through Bayesian posteriors when rewards are stable; why stale evidence slows learning when the reward distribution changes; and whether forgetting mechanisms such as windows, discounting, and change detection can fix that failure mode.

1
Build the stationary baseline

Use a standard Bernoulli multi-armed bandit to verify the basic strength of TS.

2
Create a non-stationary shock

Reverse the best arm halfway through the horizon and observe whether old posterior evidence becomes harmful.

3
Add active forgetting

Compare sliding windows, discounting, non-stationary UCB baselines, and adaptive windows.

4
Stress-test the conclusion

Vary windows, discount factors, environment shapes, and Adaptive parameters instead of reporting a single lucky configuration.

1. Problem Setting: Why Study Bandits

The multi-armed bandit is a canonical online decision problem. At each round, the learner chooses one action, observes only that action's random reward, and never sees the counterfactual rewards of the unchosen actions. The learner must explore uncertain actions while exploiting the action that currently looks best: the exploration-exploitation trade-off.

In a stochastic finance setting, the same structure can be read as dynamically allocating trial opportunities across strategies, assets, signals, or trading rules. Bernoulli bandits keep the experiment simple while making the differences among TS, UCB, sliding windows, and discounting easy to inspect.

2. Algorithm Intuition: What Each Method Tries to Solve

Thompson Sampling

Maintains a Beta posterior for each arm, samples once from each posterior, and selects the arm with the highest sampled value. Uncertainty directly becomes randomized exploration, which is why TS learns efficiently in stationary environments.

Epsilon-Greedy / UCB1

Epsilon-Greedy explores with a fixed probability; UCB1 encourages uncertain arms through confidence bounds. They serve as classic baselines for assessing whether Bayesian posterior sampling is more effective.

DS-TS / D-UCB

Discounted methods downweight old observations so the learner focuses on recent evidence. Too much discounting loses useful samples; too little discounting reacts slowly, so these methods are parameter-sensitive.

SW-TS / SW-UCB

Sliding-window methods keep only the most recent W observations. The mechanism is clear and reacts directly to abrupt changes, but very small windows increase estimation variance.

Adaptive-SW-TS

A heuristic extension in this project: switch between small and large windows using change detection. It illustrates automatic adaptation, but ablations are needed to test whether it truly beats a fixed window.

Random Baseline

The random policy is not a serious competitor; it is a lower-bound reference. It confirms that the environment contains learnable signal and that better algorithms are actually extracting reward structure.

4. Metrics: Why Regret, Reward, and Optimal-Arm Rate Matter

Cumulative regret

Measures the accumulated gap between the chosen arm and the best arm at each round. It is the core metric here; in non-stationary settings, adaptation speed near change points appears directly in the regret slope.

Cumulative reward

Explains performance from a payoff perspective and complements regret. It is useful for intuition, but the primary algorithm ranking is based on regret.

Optimal-arm rate

Tracks how often the current best arm is selected. It is intuitive, but it cannot decide the conclusion alone because an algorithm may recover later after losing substantial regret around the change point.

SEM / repeated runs

Thirty repeated runs reduce random fluctuation. SEM helps judge whether differences are stable instead of mistaking one lucky run for an algorithmic advantage.

5. Reporting Claims: How to Phrase the Results

  • In the stationary environment, standard Thompson Sampling has the lowest regret, showing that Bayesian posterior sampling balances exploration and exploitation efficiently.
  • In the abrupt non-stationary environment, standard TS suffers from posterior inertia; SW-TS adapts faster by retaining only recent observations.
  • Window size creates a bias-variance trade-off: small windows react quickly but estimate noisily, while large windows are stable but may react slowly. In this setup, W=2000 is best.
  • DS-TS is sensitive to the discount factor. A small gamma over-forgets, while a large gamma approaches standard TS, so stronger discounting is not automatically better.
  • Adaptive-SW-TS is a reasonable extension direction, but it does not stably outperform fixed large-window SW-TS in these experiments. It should be reported as a heuristic extension, not a proven improvement.

Reproducibility

Experiment Reproduction Pipeline

The dashboard keeps summary CSVs and final figures. The original experiment codebase lives in the parent directory at `00_extracted/ts_nonstationary_project`.

1

Environment setup

Create a Python virtual environment, install dependencies, and install the experiment package in editable mode.

2

Full run

Use `--experiment all --horizon 10000 --n-runs 30` to generate all summary tables and figures.

3

Result archive

Copy summary files from `results/*.csv` and figures from `results/figures/*.png` into this dashboard directory.

4

Report usage

Prioritize the non-stationary regret figure, window sensitivity, the three extended sweep figures, and the final ranking tables.

cd "00_extracted/ts_nonstationary_project"
python -m ts_bandits.run_all --experiment all --horizon 10000 --n-runs 30 --output-dir results
